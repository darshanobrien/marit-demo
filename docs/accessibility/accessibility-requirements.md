# Accessibility Requirements

## Target

Marit targets WCAG 2.2 AA.

## Keyboard Access

- All interactive controls must be reachable by keyboard.
- Focus order must follow visual and logical order.
- Focus indicators must be visible.
- Modals, menus, and tabs must follow expected keyboard patterns.

## Semantic Structure

- Use semantic landmarks for header, navigation, main content, and complementary areas.
- Use headings in a logical order.
- Use buttons for actions and links for navigation.
- Use native form controls where practical.

## Forms

- Every field must have an accessible label.
- Help text should be associated with fields.
- Validation messages must identify the issue and be connected to the relevant field.
- Required fields must be indicated in text.

## Status, Risk, and Scores

- Do not rely on colour alone.
- Include text labels for statuses and risk levels.
- Use accessible descriptions for charts or score visuals.

## Language and Localization

- Set the document language based on locale.
- Ensure French and English text fit in layouts.
- Avoid truncating critical labels or messages.

## Testing

Use a mix of:

- Keyboard walkthroughs.
- Automated accessibility checks.
- Screen reader spot checks for key flows.
